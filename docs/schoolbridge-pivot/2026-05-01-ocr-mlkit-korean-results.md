## 4. Quality Gate (Android 구현 기준)

```
overall_score = text_quality × 0.62 + pattern × 0.38   (테이블 구조 없을 때)
overall_score = text_quality × 0.50 + pattern × 0.35 + table × 0.15  (테이블 있을 때)

auto_pass 임계값: 0.80  ← 실험 최고치(0.82) 기반으로 조정
```

현재 실험 결과 overall 최고 **0.8242** — 임계값 0.80 기준 정상 범위.  
임계값 미달 시 OcrActivity에서 "원문 확인 필요" 경고 + 재촬영 / 그래도 전송 선택 제공.

## 5. ML Kit 채택 근거 — 왜 다른 OCR을 드랍했는가

강사님 피드백 "OCR 정확도를 믿기 어렵다"에 대한 실험 기반 반론.

| 엔진 | 탈락 이유 |
|------|----------|
| EasyOCR | CER 98.2% — 한국어 인식 실질적으로 불가 |
| Tesseract | CER 96.8% — 동일 |
| PaddleOCR | Python 3.13 + numpy 2.x 환경 호환 오류 — 실행 자체 불가 |

오픈소스 3종이 한국어에서 한계가 명확한 반면, **ML Kit Korean은 Google이 실제 서비스(Lens, Translate 등)에 배포한 온디바이스 모델**이다. 별도 파인튜닝 없이 F1 0.95+ 달성 — 이미 대규모 한국어 데이터로 검증된 모델이기 때문이다.

따라서 "OCR을 시도했다가 포기"가 아니라 **"실험으로 한계를 확인하고, 검증된 엔진만 선택적으로 채택"** 한 것이다. 보조 기능으로 포지셔닝하더라도 수치 근거가 있다.
